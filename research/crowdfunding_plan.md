# ショート動画 × クラウドファンディング — 事業計画プラン

## 概要
TikTok型ショート動画 × CAMPFIRE型クラウドファンディングを融合したプラットフォームの事業立ち上げ計画。
調査・検討フェーズから西野さんへのピッチ準備までを体系的に進める。

### 対象取引類型（改訂済み — 2026-02-14）

- **プロジェクト応援**（All or Nothing）: 唯一の取引類型
- **対象外**: 投資型（エクイティ）/ 融資型 / 寄付型 / **投げ銭**

> **設計判断**: 「投げ銭」と「CF支援」の区別を廃止し、全ての金銭移動を「プロジェクト応援」に統一。
> ユーザーは金額を選んで応援するだけ。金額に応じたティアが自動適用される。
> これにより (1) UXの単純化、(2) Apple IAP論点の消滅（P2P送金審査が不要）、
> (3) 全取引からPF手数料を徴収可能、(4) 決済フローの一本化 を実現。
> 詳細: research/phase5_revenue_and_ux_design.md

### プランのスコープ
本プランは **調査→ビジネスモデル確定→ピッチ準備** までをカバーする。
MVP開発・実測検証・実際の決済事業者契約は次フェーズのスコープとする。
Phase 5のGo/No-Go判定は調査・仮説ベースであり、MVP後に実測データで再検証する前提。

## 実行方針

**Claude Code のエージェントチーム機能（Task tool）を活用し、独立した調査タスクを並列実行する。**

各フェーズ内の独立した調査項目は、サブエージェント（WebSearch / Explore / general-purpose）を同時起動して並列処理し、効率を最大化する。依存関係のあるタスクのみ逐次実行とする。

### データ品質基準
- 一次情報（公式発表、決算資料、官公庁統計）を優先
- 二次情報（メディア記事、ブログ）は一次情報で裏取りできる場合のみ採用
- 情報の更新日を必ず記録し、2年以上古いデータには注釈を付ける
- データ欠損時の対応: トラフィック推計・第三者調査レポートで補完し、信頼度をA（一次情報）/B（裏取り済み二次情報）/C（推計値）でランク付け。C評価のデータには感度分析を必須とする

## Phase 1: 法規制・コンプライアンス調査（最優先）

> ビジネスモデルの根幹に関わるため、他のすべてに先行して実施。

以下を **並列実行（4エージェント同時起動）**:

| エージェント | 調査内容 |
|-------------|---------|
| Agent A | 資金移動業の登録要件 — All or Nothingのエスクロー（支援金一時預かり）が資金移動業に該当するか、登録手続き・コスト・期間 |
| Agent B | CAMPFIREの法的構造 — どのライセンスで運営しているか、決済代行の仕組み、過去の法的トラブル事例 |
| Agent C | 投げ銭・クラファンの法的分類 — 特定商取引法の適用範囲、投げ銭の贈与/役務対価の分類、KYC要件 |
| Agent D | 追加法令調査 — 個人情報保護法（動画・支援者データ）、AML/CFT（マネロン対策）、未成年者保護（年齢確認・利用制限）、著作権・コンテンツ権利処理、景品表示法 |

**完了条件**:
- 法令論点マトリクス（論点・根拠条文・該当機能・必要対応）の完成
- 自社に必要なライセンス・届出の一覧と取得コスト・期間の明記
- ビジネスモデルに対する法的制約の一覧
- 必要な運用体制の概要（投稿審査、モデレーション、通報対応、返金オペレーション等の論点洗い出し）

**ゲート条件（Phase 4 開始前に必須）**:
弁護士への相談を実施し、少なくとも以下について法的見解を確認すること:
- 資金移動業への該当性とライセンス要否
- AML/CFT対応の要否と範囲
- 推奨される法的スキーム（決済代行経由等）

**規制重課時のピボット分岐**:
- 許認可取得に6か月以上かかる場合 → 決済代行経由のスキームに変更、または B2B SaaS型（既存クラファンへの動画機能提供）へピボットを検討
- ピボット判断はPhase 1完了+弁護士見解の時点で実施
- ピボット発生時の再実行対象: Phase 2（競合・市場をB2B軸で再調査）、Phase 3（ターゲットをクラファン事業者に変更）、Phase 5（収益モデル・Go/No-Go基準を再設計）、Phase 6（ピッチ先を変更）

### 概算期間
- AI調査: 1日
- 弁護士相談: 1〜2週間（アポ確保〜回答受領）

## Phase 5: ビジネスモデル確定

> Phase 1〜4 の結果を統合して決定（手動検討）。

### 必須入力（全て揃うまで開始しない）
- ✅ Phase 1: 法令論点マトリクス + 弁護士見解
- ✅ Phase 2: 競合比較表 + 市場規模推計
- ✅ Phase 3: ターゲット定義 + 検証計画 + インタビュー要約（主要仮説の支持/反証。インタビュー未完了の場合はAI調査ベースの仮説で進め、後日インタビュー結果で再検証）
- ✅ Phase 4: 決済基盤評価表

### 収益設計
- プラットフォーム手数料率: **10%**（暫定。全「応援」取引に適用）
- ~~投げ銭手数料率~~: 廃止（投げ銭を取引類型から除外）
- サブスク要素の有無: 未定（プレミアムクリエイターツール等、成長期に検討）
- 決済手数料: PAY.JP 2.78%（クリエイター負担 or PF負担を決定する必要あり）
- Apple/Google手数料: **0%**（物理的成果物を持つCFプラットフォームとしてIAP不要）

### コスト構造の分解
- 決済手数料（決済代行への支払い）
- チャージバック・返金コスト（All or Nothing返金のタイムラグ含む）
- 不正対策・審査運用コスト
- 動画インフラ（ホスティング・CDN・エンコード）
- 人件費（初期のクリエイター獲得・動画制作支援）
- ユーザー獲得コスト（マーケティング・インセンティブ）

### ユニットエコノミクス（3シナリオ）
| シナリオ | 内容 |
|---------|------|
| 悲観 | 低GMV・高コスト前提 |
| 基準 | 現実的な想定 |
| 楽観 | 高成長前提 |

各シナリオで以下を**支援者側・クリエイター側それぞれ**で算出:
- CAC（顧客獲得コスト）— 支援者: マーケ費/支援者数、クリエイター: 獲得・制作支援費/クリエイター数
- LTV（顧客生涯価値）— 支援者: 生涯支援総額×手数料率、クリエイター: 生涯GMV×手数料率
- 粗利率（プラットフォーム全体）
- 損益分岐点のGMV
- 黒字化までの期間

### 月次資金繰り計画（12か月分）
- 月次バーンレート
- 必要運転資金（返金タイムラグ考慮）
- 最悪ケースでの資金ショート時期
- 必要な初期資金の総額

### Go/No-Go 判定基準
| 条件 | 閾値 |
|------|------|
| 法務未解決の重大論点 | 0件 |
| 想定粗利率（基準シナリオ） | 30%以上 |
| CAC回収期間・支援者側（基準シナリオ） | 12か月以内 |
| CAC回収期間・クリエイター側（基準シナリオ） | 12か月以内 |
| 資金ショートまでの期間（悲観シナリオ） | 18か月以上 |

> Go/No-Go判定は2段階で運用する:
> - **暫定Go**: AI調査ベースの仮説値で判定。ピッチ準備（Phase 6）への着手を許可
> - **確定Go**: インタビュー結果・MVP実測を反映して再検証（次フェーズ）。本格投資へのコミットはここで判断

閾値未達の場合: ピボット（モデル変更）or 追加資金調達の検討

### 概算期間: 2〜3日（意思決定を含む）

## リスク管理

| リスク | 影響 | 対策 |
|--------|------|------|
| 資金移動業の登録が必要 | ローンチが半年〜1年遅延 | 決済代行経由でライセンス不要の設計を検討 |
| 法規制で重大な障壁 | ビジネスモデル変更が必要 | Phase 2以降の方針を見直し |
| 西野さんからの反応なし | 初期ユーザー獲得戦略の変更 | 代替候補リスト（3〜5名）を事前準備、2週間でGo/No-Go判定 |
| 類似サービスが海外に存在 | 差別化の再定義が必要 | その成否要因を分析し、学びを取り込む |
| クリエイター供給不足 | 2面市場が成立しない | 初期は自社制作支援で供給を担保、特化カテゴリで集中 |
| Go/No-Go基準未達 | 事業計画の見直し | ピボット案を事前に2〜3パターン検討 |
| Apple審査でIAP必須と判定 | 手数料構造が変わる | 全PJに物理的成果物を要件化しKickstarter同等の構造を維持。投げ銭概念を廃止し審査論点を削減 |
| PF手数料10%での収益不足 | 初期赤字が長期化 | 損益分岐GMV ¥4,000〜6,500万/月。成長期にプロモーション掲載・プレミアムツールで収益源を追加 |
