## 1. Stripe Connect（日本）の手数料

### 1.1 標準料率（2025-2026年現在）

| 決済方法 | 手数料率 | 備考 |
|---------|---------|------|
| クレジットカード（国内Visa/Mastercard/AMEX） | **3.6%** | 固定料率、トランザクション毎の定額加算なし |
| JCB | **3.6%** | 同上 |
| 海外カード | **3.6% + 1.5%**（国際手数料） | 通貨変換時はさらに+1% |
| コンビニ決済 | **¥150〜¥500/件** | 件あたり固定、料率ではない |
| 銀行振込 | **1.5%** | カードより安い |

> **注**: 2023年10月のインボイス制度導入後、Stripeの手数料自体にも消費税10%が課税される。つまり実質 3.6% × 1.1 = **3.96%** が実コスト。

### 1.2 Custom vs Express アカウントの違い

| 項目 | Standard | Express | Custom |
|------|----------|---------|--------|
| **ユーザー体験** | Stripe管理画面あり | Stripe簡易画面あり | 完全非表示（自社UI） |
| **本人確認** | ユーザー自身 | Stripeが処理 | プラットフォームが処理 |
| **追加コスト** | なし | Connect手数料 | Connect手数料 |
| **決済手数料** | 3.6%（同一） | 3.6%（同一） | 3.6%（同一） |
| **Connect手数料** | なし | $2/アクティブアカウント/月 | $2/アクティブアカウント/月 |
| **推奨用途** | 個人利用 | マーケットプレイス | フルカスタムプラットフォーム |

> **重要**: Express/Customでは、アクティブなConnectアカウント1つにつき月額$2（約¥300）が追加で発生する。クリエイター1万人なら月$20,000（約¥300万）。

### 1.3 手数料3%まで下げられるか？

**結論: 個別交渉で可能だが、条件は厳しい。**

- Stripeは月間処理額 **$80,000（約¥1,200万）以上** でカスタム料金の交渉に応じる
- 大規模事業者（月間数億円規模）では **2.5〜3.0%** まで下がった事例がある
- 交渉で有利になる条件:
  - 安定した高ボリューム
  - 低チャージバック率
  - 長期コミットメント（年間契約）
  - Interchange Plus（IC+）モデルへの移行
- **現実的な最低ライン**: 日本市場では **3.0〜3.25%** が限界（Visa/Mastercard）
- JCBは交渉しても下がりにくい（カード会社側のIC率が高い）

### 1.4 ボリュームディスカウント

| 月間処理額 | 想定料率 | 交渉難易度 |
|-----------|---------|-----------|
| 〜¥500万 | 3.6%（標準） | 交渉不可 |
| ¥500万〜¥3,000万 | 3.3〜3.5% | 営業チーム対応 |
| ¥3,000万〜¥1億 | 3.0〜3.3% | IC+モデル提案あり |
| ¥1億以上 | 2.5〜3.0% | エンタープライズ契約 |

## 2. TikTok Shopの決済処理

### 2.1 決済の仕組み

- TikTok Shopは **自社決済基盤** を使用（Stripeではない）
- 支払い方法: Visa/Mastercard/AMEX/Discover、PayPal、Apple Pay、Google Pay、Klarna、Affirm
- TikTokが決済を処理し、売上金をセラーの銀行口座に振り込む
- 振込タイミング: 配送完了後1〜31日

### 2.2 TikTok Shopの手数料体系（2025年現在）

| 項目 | 料率 | 備考 |
|------|------|------|
| **リファラルフィー（標準）** | **6%** | カテゴリにより5〜6% |
| **新規セラープロモ（30日間）** | **3%** | オンボーディング60日以内に初売上達成が条件 |
| **返金管理手数料** | リファラルフィーの20%（最大$5） | 返金時のみ |
| **出金手数料** | $0.05/回 | 最低出金額$2 |

### 2.3 低手数料の仕組み

TikTok Shopが低手数料（実質6%のみ）を実現できる理由:

1. **物理商品のマーケットプレイス** → Apple/Google IAPの30%が適用されない
2. **巨大なトラフィック** を既に持っているため、広告費不要でセラーに集客を提供
3. **決済処理を自社運営** → Stripe等の外部決済手数料を内部コスト化
4. **戦略的価格設定** → 市場シェア獲得のため利益度外視の料率設定（中国ByteDance資本）
5. **物流統合** → フルフィルメントを提供することでスケールメリット

### 2.4 Apple/Google IAPの回避方法

TikTok Shopは「回避」ではなく「**正当に非該当**」:

- Apple/Google IAPは **デジタルコンテンツ・サービス** にのみ適用
- TikTok Shopは **物理商品のEC** であり、IAP対象外
- Amazon、DoorDash、Uber等と同じ分類
- アプリ内でApple Pay（決済手段として）を使っても、IAP手数料は発生しない

### 3.2 PAY.JP Platform（Stripe Connect相当）

PAY.JPには **PAY.JP Platform** というStripe Connect相当の機能がある。

- マーケットプレイス・プラットフォーム向け
- テナント（クリエイター）ごとにサブアカウント作成可能
- 売上分配・送金を自動化
- 手数料: 上記プランの決済手数料 + Platform利用料（要問合せ）

**PAY.JPがStripeより有利な点**:
- **決済手数料が安い**（スタンダードでも3.3% vs 3.6%）
- 国内企業のためサポートが日本語
- JCB対応がネイティブ

**PAY.JPがStripeより不利な点**:
- 海外展開が困難
- Connect相当機能のエコシステムが小さい
- ドキュメント・ライブラリの充実度でStripeに劣る

## 4. Apple/Google 30%税の回避に関する法的構造

### 4.1 クラウドファンディングの「Pledge」はIAP対象か？

**結論: 「物理的リターン」があればIAP対象外。デジタルのみだとIAP対象になるリスクがある。**

#### Apple App Storeガイドライン上の分類

| リターンの種類 | IAP必須？ | 根拠 |
|--------------|----------|------|
| 物理商品（グッズ、限定品等） | **不要** | 物理商品はIAP対象外 |
| アプリ外で消費されるサービス | **不要** | イベント参加、体験等 |
| デジタルコンテンツ（限定動画、スタンプ等） | **必要（30%課税）** | デジタル商品・サービスはIAP必須 |
| 投げ銭（クリエイターへの送金） | **グレーゾーン** | 下記で詳述 |
| 寄付（認定NPO宛） | **不要** | Apple Pay利用可、手数料免除 |

#### 投げ銭の分類問題

- Appleガイドライン上、アプリ内での「Tip（チップ）」はIAPを使用して行う必要がある
- つまり **投げ銭にはAppleの30%（小規模事業者は15%）が適用される**
- Patreonが実際にiOSアプリでIAPを導入した結果、iOS経由の支払いには30%が追加された

### 4.2 適用可能な免除・回避策

#### (A) 物理商品マーケットプレイスとしての設計

**Kickstarterモデル**: プロジェクトのリターンが物理商品（アルバムCD、グッズ、体験型イベント等）であれば、IAP不要。
- Kickstarterアプリは **Apple Pay**（IAPではない）を使用して決済
- 「プロジェクトへのPledge = アプリ外で履行される商品/サービスの予約購入」と分類
- これが成立するには、**リターンの主体が物理商品またはアプリ外サービス** である必要がある

#### (B) リーダーアプリ免除（Reader App Entitlement）

- 「リーダーアプリ」（音楽、動画、書籍等のコンテンツ閲覧アプリ）は外部決済リンクを設置可能
- **クラウドファンディングはリーダーアプリに該当しない** ため、この免除は使えない

#### (C) マーケットプレイス免除（EU DMA対応）

- EUのDigital Markets Act（DMA）対応で、EU圏内では代替決済が可能
- ただし、**日本市場には適用されない**（2026年2月現在）
- Apple独自の「Alternative App Marketplace」エンタイトルメントも現状EU限定

#### (D) Webベースチェックアウト（Safari/WebViewリダイレクト）

**2025年5月の米国裁判所判決後の状況（米国のみ）**:
- iOS内に外部決済リンク（1つ）を設置可能
- Apple の「迂回手数料」は **0%**（ただしAppleが控訴中）
- 外部Webチェックアウトに飛ばせば、Stripeの3.6%のみで済む

**日本での状況**:
- **米国判決は日本に直接適用されない**
- 日本のApp Storeでは依然としてIAPが必須（デジタルコンテンツの場合）
- ただし、Safari経由のWebチェックアウトへの「情報提供リンク」は設置可能になりつつある
- Apple側も段階的に規制を緩和する動きがある

#### (E) Google Playの状況

- 2025年10月以降、米国ではGoogle Play外の決済手段も許可
- 日本では **物理商品・サービスはGoogle Play Billing不要**（元々免除）
- デジタルコンテンツは依然としてGoogle Play Billing必須（15〜30%）

### 4.3 実戦的な推奨戦略

```
┌─────────────────────────────────────────────────┐
│  クラウドファンディング（購入型・物理リターン）  │
│  → IAP不要、自社決済OK（3〜4%で済む）           │
│  → Kickstarterと同じApple Payモデル              │
├─────────────────────────────────────────────────┤
│  投げ銭（デジタル・純粋なTip）                    │
│  → IAP必須（Apple 30% + 決済手数料）             │
│  → または Web経由チェックアウトで回避            │
│  → iOSアプリでは投げ銭機能を非表示にし、         │
│    Webブラウザ版でのみ提供する手法もある          │
├─────────────────────────────────────────────────┤
│  投げ銭の代替設計                                │
│  → 「応援購入」として物理リターン付きにする       │
│    （例: お礼メッセージカード送付 + 投げ銭）     │
│  → これでIAP回避できるかはAppleの審査次第        │
│  → リスクあり: Appleが「実質デジタルTip」と      │
│    判断すればリジェクトされる                    │
└─────────────────────────────────────────────────┘
```

## 5. 主要プラットフォームの実際の決済手数料

### 5.1 各プラットフォーム比較

| プラットフォーム | プラットフォーム手数料 | 決済手数料 | 合計（概算） | iOS追加コスト |
|----------------|---------------------|-----------|-------------|-------------|
| **Kickstarter** | 5% | 3% + $0.20/件（>$10）、5% + $0.05/件（<$10） | **8〜10%** | なし（物理商品Pledge） |
| **Indiegogo** | 5% | 2.9% + $0.30/件 | **約8%** | なし（物理商品Pledge） |
| **CAMPFIRE** | 12%（税別） | 5%（税込） | **約17%（税込18.7%）** | なし（物理商品Pledge） |
| **Patreon** | 8〜12%（旧プラン）、10%（2025年8月以降新規） | 2.9% + $0.30（>$3）、5% + $0.10（<$3） | **12〜15%** | iOS経由は+30%（IAP） |
| **TikTok Shop** | 5〜6%（リファラルフィー込み） | 込み | **5〜6%** | なし（物理商品EC） |

### 5.2 手数料の内訳分析

#### Kickstarter
- 決済にStripeを使用
- Stripe手数料を「決済手数料」として利用者に転嫁
- プラットフォーム手数料5%が純利益
- **All or Nothing** のため、目標未達の場合は手数料ゼロ（返金）

#### Indiegogo
- 同じくStripe使用
- InDemand（継続販売）も同じ料率
- Flexible Funding（いくらでも受取可能）も同じ5%

#### CAMPFIRE
- 決済代行は自社提携（GMO PG等を利用）
- **17%は国際的に見て高額**（Kickstarterの約2倍）
- ただしCampfireはプロジェクト支援サービスを含む
- 支援者側にもシステム利用料（2.27% + 税）が別途課金される
  - 1万円未満: ¥228 + 税
  - 1万円以上: 支援金額の2.27% + 税

#### Patreon
- 2025年8月以降の新規クリエイターは一律10%プラットフォーム手数料
- 既存のFoundersプラン（8%）は据え置き
- **iOS問題**: iOSアプリ経由の支払いにはApple IAPの30%が加算
  - $10の支援 → クリエイターは約$6.7しか受け取れない
  - Patreonはこのコストを支援者に転嫁（iOS価格が30%高い）

## 6. 結論と推奨事項

### 6.1 決済手数料の最小化

1. **初期段階（月商¥500万未満）**: PAY.JP スタンダード（3.3%）が最適。StripeのConnect機能が必要ならStripe（3.6%）
2. **成長段階（月商¥500万〜¥3,000万）**: PAY.JP ビジネス（2.78%）+ PayPay直接（1.6%）のハイブリッド
3. **スケール段階（月商¥1億超）**: Stripeとのエンタープライズ交渉（3.0%以下を目指す）またはPAY.JPエンタープライズ（2.59%）

### 6.2 Apple/Google 30%の回避

1. **購入型クラウドファンディング（物理リターン）**: 全額IAP対象外 → 自社決済でOK
2. **投げ銭**:
   - **最善策**: Webブラウザ経由でのみ提供（アプリ内では投げ銭機能を載せない）
   - **次善策**: 「応援購入」として物理リターン（お礼カード等）をセットにする
   - **最悪**: iOS/Androidアプリ内でIAP経由（30%課税）
3. **長期的**: 日本でもApple/Googleの外部決済リンク規制が緩和される可能性を注視

### 6.3 理想的なコスト構造

```
目標: クリエイターの受取率を最大化

購入型クラファン:
  支援金 100%
  - 決済手数料 2.78%（PAY.JP）
  - プラットフォーム手数料 10%（自社収益）
  = クリエイター受取 87.22%

  ※ CAMPFIREの場合は受取 約81.3%（17%+税）
  ※ Kickstarterの場合は受取 約90%（8-10%）

投げ銭（Web経由）:
  投げ銭 100%
  - 決済手数料 2.78%（PAY.JP）
  - プラットフォーム手数料 15%（自社収益）
  = クリエイター受取 82.22%

  ※ 少額（¥100-500）の場合、固定手数料の影響が大きくなる
  ※ PayPay経由なら決済手数料1.6%でさらに有利
```
