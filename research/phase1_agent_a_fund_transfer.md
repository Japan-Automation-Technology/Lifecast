## 1. 資金移動業への該当性分析

### 1.1 資金移動業の定義

資金移動業とは、銀行以外の者が「為替取引」を業として営むことを指す（資金決済法第2条第2項）。内閣総理大臣への登録が必須であり、無登録での営業は銀行法違反となる（3年以下の懲役または300万円以下の罰金）。

**為替取引の定義**（最高裁平成13年3月12日決定）:
> 「顧客から、隔地者間で直接現金を輸送せずに資金を移動する仕組みを利用して資金を移動することを内容とする依頼を受けて、これを引き受けること、又はこれを引き受けて遂行すること」

**信頼度: A**（法令・最高裁判例）

### 1.2 購入型クラウドファンディング（All or Nothing）の該当性

#### 該当する可能性が高い要素

プラットフォームが以下のフローで資金を取り扱う場合、為替取引に該当する:

```
支援者 → [プラットフォーム（一時預かり）] → クリエイター
```

- 支援者からの資金を「預かり」、手数料を天引きしてクリエイターに「送金」する構造は、隔地者間の資金移動に該当
- エスクロー（一時預かり）機能は、銀行法上の為替取引に該当する可能性がある
- All or Nothing方式では、目標未達時の返金処理も含め、プラットフォームが資金移動の主体となる

#### 金融庁の整理

2009年の金融審議会「決済に関するワーキング・グループ」報告書では、収納代行・代引きの為替取引該当性について「直ちに規制の対象とすべきではない」と整理され、将来の検討課題とされた。しかし、これはコンビニ収納代行等の国内取引を念頭に置いたものであり、クラウドファンディングのエスクローに直接適用されるかは不明確。

**信頼度: B**（弁護士解説に基づく整理）

#### 令和2年改正資金決済法（2021年5月施行）の影響

令和2年改正で「為替取引」の定義が明確化された。購入型クラウドファンディングが資金移動業に該当しない（＝適用除外となる）ためには、以下の2要件を**両方**満たす必要がある:

**要件A（債務消滅要件）**: 支援者がプラットフォーム上で支払いを完了した時点で、クリエイターに対する法律上の支払義務が消滅すること

**要件B（契約関与要件）**: プラットフォーム事業者が契約の成立に不可欠な関与を行い、受取人の同意の下に、契約内容に応じて資金を移動させること

**信頼度: A**（法令条文に基づく）

### 1.3 結論: 該当性判断

| 機能 | 該当性 | 理由 |
|------|--------|------|
| All or Nothing エスクロー | **該当する可能性が高い** | 支援者→PF→クリエイターの資金移動は為替取引の構成要件を満たす。ただし、スキーム設計により回避可能 |
| 投げ銭（直接送金型） | **該当する** | 視聴者→PF→クリエイターの資金移動は典型的な為替取引 |

**重要**: スキーム設計（決済代行・収納代行の活用）により、資金移動業登録を回避できる可能性がある（後述）。

## 2. 登録手続き

### 2.1 登録要件（資金決済法第40条）

| 要件 | 内容 | 信頼度 |
|------|------|--------|
| 法人格 | 株式会社であること（個人不可） | A |
| 財産的基礎 | 資金移動業を適正かつ確実に遂行するために必要と認められる財産的基礎を有すること | A |
| 体制整備 | 資金移動業を適正かつ確実に遂行する体制、及び法第3章の規定を遵守する体制が整備されていること | A |
| 商号 | 他の資金移動業者と同一又は類似の商号・名称を用いていないこと | A |
| 欠格事由 | 過去5年間に登録取消し、法令違反による罰金刑がないこと | A |

### 2.2 登録拒否事由

- 破産手続開始決定を受けて復権していない者
- 禁錮以上の刑に処せられ、5年未経過の者
- 資金決済法・銀行法等の違反で罰金刑に処せられ、5年未経過の者
- 過去5年間に資金移動業・資金清算業の登録・免許を取り消された者

**信頼度: A**（資金決済法第40条）

### 2.3 申請手続きの流れ

```
Step 1: 事前相談
  ├─ 「事前相談整理表」等を提出
  └─ 管轄財務局と面談

Step 2: ドラフト審査
  ├─ 申請書類すべてのドラフトを提出
  └─ 財務局による内容確認・修正指示

Step 3: 本審査（正式申請）
  ├─ 申請書類すべてを正式提出
  ├─ 登録免許税 15万円 納付
  └─ 標準処理期間: 2ヶ月
```

| 取締役等の住民票 | |

| 定款及び登記事項証明書 | |

**信頼度: A**（関東財務局公表資料、日本資金決済業協会）

| 登録免許税 | **15万円** | A |
| 行政書士・弁護士への申請代行報酬 | **440万円～（税込）** | B |
| 社内体制整備コスト（規程類作成、システム構築等） | **500万～2,000万円**（規模による） | C |

### 3.2 履行保証金（供託金）

| 項目 | 内容 | 信頼度 |
|------|------|--------|
| 最低供託額 | **1,000万円** | A |
| 算出方法 | 各営業日における未達債務額 + 還付手続費用（未達債務1億円以下の場合は5%相当） | A |
| 供託義務 | 送金サービスで預かった金額の**100%以上** | A |

#### 資金移動業の種別と送金上限

| 種別 | 送金上限額 | 供託期限 | 該当可能性 |
|------|-----------|----------|-----------|
| 第一種 | 制限なし | 2営業日以内 | 低（高額送金不要） |
| 第二種 | **100万円以下** | 3営業日以内 | **高（CF・投げ銭に適合）** |
| 第三種 | 5万円以下 | 3営業日以内 | 投げ銭のみなら可能性あり |

**本プラットフォームの場合**: クラウドファンディングの支援金額を考慮すると、**第二種資金移動業**（100万円以下）が最も現実的。

### 3.3 維持費用（年間ランニングコスト）

| 項目 | 概算金額 | 信頼度 |
|------|----------|--------|
| 日本資金決済業協会 年会費 | 約50万～100万円 | C |
| コンプライアンス担当者人件費 | 500万～800万円/年 | C |
| 外部監査費用 | 100万～300万円/年 | C |
| システム維持・AML対応 | 200万～500万円/年 | C |
| 履行保証金の維持（機会コスト） | 供託額に応じて変動 | C |

### 3.4 保全方法の選択肢

供託以外に以下の方法で履行保証金を保全可能:

1. **履行保証金保全契約**: 銀行等との間で締結（内閣総理大臣の承認要）
2. **履行保証金信託契約**: 信託会社等との間で締結（内閣総理大臣の承認要）

これらにより、現金供託を回避し資金効率を高めることが可能。

**信頼度: A**（資金決済法第43条～第45条）

## 5. 回避策: 決済代行の活用

### 5.1 決済代行スキームによる回避の可能性

**結論: 適切なスキーム設計により、資金移動業登録を回避できる可能性がある。**

#### 回避の基本構造

```
【資金移動業に該当するスキーム（NG）】
  支援者 → PF口座（預かり） → クリエイター口座
  ※PFが資金移動の主体

【決済代行スキーム（回避可能）】
  支援者 → 決済代行業者（Stripe等） → クリエイター口座
  ※PFは決済代行業者に対し決済処理を委託するのみ
  ※PFは資金に直接触れない
```

#### 令和2年改正法に基づく適用除外の2要件

回避が認められるためには、以下の**両方**を満たす必要がある:

**要件A（債務消滅要件）**:
- 支援者がプラットフォーム上で支払いを完了した時点で、クリエイターに対する法律上の支払義務が消滅すること
- 利用規約で「決済代行業者への支払いをもって支援者の支払義務は完了する」旨を明記

**要件B（契約関与要件）**:
- プラットフォーム事業者が契約の成立に不可欠な関与を行うこと
- クラウドファンディングのプロジェクト審査・掲載承認、All or Nothing判定、リターン管理等がこれに該当

### 5.2 Stripe Connectの活用

#### Stripe Connectの構造

Stripe Connect は、プラットフォーム型ビジネス向けの決済基盤で、以下の特徴を持つ:

- Stripe自身が資金決済法上の登録を保有（Stripe Japan株式会社は資金移動業者として登録済み）
- プラットフォーム事業者は資金に直接触れず、Stripeが支援者からの受領とクリエイターへの送金を処理
- Stripe手数料: 決済金額の3.6%（標準）

#### 法的整理

```
支援者 → [Stripe（決済処理・資金保全）] → クリエイター
            ↑
          PF（決済指示のみ）
```

- PFは「決済代行の委託者」であり、資金移動の主体ではない
- Stripeが資金移動業者として資金保全義務を負う
- PFは資金決済法上の登録不要

#### 注意点

- **利用規約の設計が極めて重要**: 上記2要件を満たす規約設計を弁護士と共に行う必要がある
- **Stripeの利用制限**: Stripe の利用規約上、クラウドファンディングが制限業種に含まれる場合がある。事前にStripeへの確認が必須
- **All or Nothing の資金滞留**: 目標達成判定までの期間（通常30～90日）、資金がStripe上に滞留する。Stripeの規約上これが許容されるか要確認
- **投げ銭**: 都度課金の投げ銭はStripe Connectの標準的なユースケースであり、比較的問題が少ない

**信頼度: B**（弁護士解説・Stripe公式情報の複合）

### 5.3 他の決済代行サービス

| サービス | 特徴 | クラファン対応 |
|----------|------|--------------|
| Stripe Connect | プラットフォーム型に最適化 | 要確認 |
| GMO Payment Gateway | 国内大手、CF実績あり | CAMPFIRE等が利用 |
| PAY.JP（PAY株式会社） | シンプルなAPI | 要相談 |
| SBペイメントサービス | 多様な決済手段 | 要相談 |

**信頼度: C**（一般的な市場情報）

## 6. 収納代行との違い

### 6.1 基本的な違い

| 項目 | 収納代行 | 決済代行 | 資金移動業 |
|------|----------|----------|-----------|
| 定義 | 債権者の代わりに代金を受領する代行サービス | 決済処理全般を代行するサービス | 為替取引を業として行う |
| 資金の流れ | 債務者→収納代行→債権者 | 購入者→決済代行→販売者 | 送金人→資金移動業者→受取人 |
| 法的位置づけ | 法令上の明確な定義なし（判例・行政解釈による） | 法令上の明確な定義なし | 資金決済法で規制 |
| 登録要件 | 原則不要（ただし2025年改正でクロスボーダーは規制対象） | 原則不要 | **内閣総理大臣への登録必須** |
| 典型例 | コンビニ収納代行、公共料金収納 | クレジットカード決済代行 | PayPay送金、LINE Pay送金 |

### 6.2 収納代行スキームの適用可能性

#### 収納代行が為替取引に該当しないとされる根拠

- 収納代行は「代金の受領を代行する」にすぎず、資金の「移動」ではない
- 原因取引（商品売買等）と不可分一体であり、独立した送金サービスではない
- 2009年金融審議会報告書で「直ちに規制対象とすべきではない」と整理

#### 本プラットフォームへの適用

**All or Nothing エスクローの場合**:
- プラットフォームがクリエイターの「代理人」として支援金を「受領」する構成を取れば、収納代行として整理可能
- ただし、目標未達時の「返金」が為替取引に該当するリスクが残る
- All or Nothing特有の「条件付き送金」は、単純な収納代行の範囲を超える可能性がある

**投げ銭の場合**:
- クリエイターの代理人として投げ銭を受領する構成は比較的取りやすい
- ただし、ポイント制でない直接送金型は為替取引と判断されるリスクが高い

### 6.3 2025年改正の影響

令和7年（2025年）改正資金決済法により、**クロスボーダー収納代行**は原則として資金移動業の規制対象となった。

| 項目 | 改正前 | 改正後 |
|------|--------|--------|
| 国内完結の収納代行 | 規制対象外 | **変更なし（規制対象外のまま）** |
| クロスボーダー収納代行 | グレーゾーン | **原則として資金移動業の登録必要** |
| 適用除外 | - | エスクローサービス、取引PF提供者によるもの等 |

**重要**: 国内のみで完結する購入型クラウドファンディングの場合、2025年改正の直接的な影響は限定的。ただし、海外クリエイターへの送金を行う場合は、クロスボーダー収納代行として規制対象となる可能性がある。

**信頼度: A/B**（改正法条文・弁護士解説の複合）

## 7. 投げ銭機能の法的整理

### 7.1 直接送金型の問題

視聴者がクリエイターに直接金銭を送る構造は、為替取引に該当し資金移動業の登録が必要。

### 7.2 回避スキーム

#### 方法1: 前払式支払手段（ポイント・コイン方式）

```
視聴者 → [現金でコインを購入] → [コインをクリエイターに贈る] → クリエイター
※ コインは現金ではないため為替取引に該当しない
※ クリエイターへの換金はPF内での精算処理
```

**義務**:
- 未使用残高1,000万円超で供託義務（前払式支払手段発行者として）
- 8項目の情報表示義務
- 定期報告義務
- サービス終了時の払戻義務

**注意**: コインが直接現金化できる場合（払い戻し可能な場合）、実質的に為替取引と判断されるリスクがある。コインの「換金不可」設計が重要。

**実例**: SHOWROOM（アイテム→評価ポイント→報酬の間接的構造）

#### 方法2: 決済代行経由

```
視聴者 → [Stripe等の決済代行] → クリエイター
※ PFは決済指示のみ
```

Stripe Connectを利用し、PFが資金に触れない構造。

**信頼度: B**（弁護士解説に基づく）

## 8. 推奨アクション

### 8.1 短期（MVP段階）: 決済代行活用による登録回避

| 優先度 | アクション | 備考 |
|--------|-----------|------|
| 1 | **Stripe Connectの利用可否をStripe社に確認** | CF・投げ銭の両方について |
| 2 | **利用規約を弁護士と設計** | 適用除外2要件（債務消滅・契約関与）を満たす規約 |
| 3 | **投げ銭はポイント/コイン方式を採用** | 前払式支払手段として設計し、為替取引該当を回避 |
| 4 | **金融庁・財務局への事前照会** | ノーアクションレター制度の活用を検討 |

### 8.2 中期（スケール段階）: 自社登録の検討

事業規模拡大に伴い、決済代行手数料の負担が大きくなった場合や、より柔軟な資金フローが必要になった場合は、第二種資金移動業の登録を検討。

### 8.3 コスト比較

| 項目 | 決済代行活用 | 自社で資金移動業登録 |
|------|-------------|---------------------|
| 初期費用 | ほぼゼロ（開発費のみ） | 1,000万～3,000万円 |
