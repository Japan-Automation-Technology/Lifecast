# Agent K: クリエイター供給側の分析レポート

**調査日: 2026年2月14日**

---

## 1. クリエイター獲得チャネル

### 1-1. 既存PFの初期クリエイター獲得戦略

**CAMPFIRE（2011年4月）**
- 当初は審査厳格・手数料20%でプロ寄りクリエイター限定
- 2016年の大転換: 手数料20%→5%に引下げ、審査基準緩和、All-In方式導入
- 「5,000万円のPJ1つより、5万円のPJ1,000個」を志向
- 結果: 累計10万件超、総支援額960億円超、支援者1,200万人超

**Makuake（2013年8月）**
- 「クラウドファンディング」でなく「応援購入」を標榜
- キュレーター制度: 専属担当者がPJページ作成を支援
- メーカー・製品開発者に特化したBtoB的営業チャネル

**READYFOR（2011年3月）**
- 日本初のクラファン。NPO・社会貢献に特化
- 積極的な営業活動で直接PJオーナーを獲得

> 信頼度: B

### 1-2. 西野亮廣のクラファン実績とコミュニティ

| 項目 | データ |
|------|--------|
| クラファン支援総額 | **8億円超** |
| オンラインサロン会員数 | 約24,300名（2025年4月）。ピーク時7万人超 |
| 月額会費 | 980円 |

**西野のクラファン哲学:**
- クラウドファンディングは「信用を両替する機械」
- 支援者の大多数は「会ったことがある人」「腹を割って話したことがある人」
- 「人に会いに行く」基本活動が最重要

**プロセスエコノミー実践事例:**
- 絵本制作費をクラファン調達、看板を出資者の広告に活用
- 映画公開前に脚本を販売
- ミュージカル版は4.5億円規模の制作費をプロセス販売+クラファンで調達

> 信頼度: B（西野本人ブログ・記事ベース）

### 1-3. 西野サロン経由の初期獲得戦略への示唆

| 強み | リスク |
|------|--------|
| 24,300人のコア層は支援意欲が極めて高い | 会員数はピークの1/3に減少中 |
| クラファン文化が浸透したコミュニティ | 西野個人への依存度が高い |
| メンバー同士の自主的PJ立ち上げ実績あり | サロン外への拡散力は限定的 |
| 「プロセスエコノミー」の実践者集団 | 動画制作スキルの分布は不明 |

**推奨:**
1. 西野サロンは「初期支援者（需要側）獲得チャネル」として位置づけるべき
2. サロン内「自主クラファン経験者」（推定数百名）にショート動画PJ起案を働きかけ
3. 西野本人にはアドバイザー/初期PJ起案者として関与を打診

## 2. 動画制作のハードル

### 2-1. 制作時間・コスト

| 項目 | データ |
|------|--------|
| 個人クリエイター（自作） | 15秒: 30分~1時間、1分: 2~3時間 |
| フリーランスへの外注 | 1本 1,000~15,000円（編集のみ） |
| 制作会社への外注 | 1本 3~10万円 |
| スマホのみ制作 | 十分に可能（TikTokアプリ内で完結） |

> 信頼度: B

### 2-2. 参入障壁

**低い障壁:**
- TikTokアプリ内で撮影から編集まで完結可能
- 高価な機材不要、スマートフォンで十分
- 15秒の短尺から始められる

**残る障壁:**
- 「カメラの前で話す」心理的抵抗
- 継続的なネタ出し・企画力
- サムネイル設計やフックの知識不足

### 2-3. AI支援ツールの現状（2025年~）

| ツール | 特徴 |
|--------|------|
| Canva | テキストから動画自動生成、AIナレーション・字幕自動生成 |
| Mootion | テキスト/スクリプトからショート動画自動生成 |
| Vrew | AI動画編集、自動字幕生成、ショート動画最適化 |
| invideo | テキスト入力でショート動画生成、フェイスレス対応 |

> 信頼度: A

### 2-4. PFへの示唆

- **動画テンプレート機能は必須**: 「PJ紹介」「進捗報告」「お礼」等のテンプレート
- **フェイスレス動画の許容**: 顔出し障壁向けにスライドショー+ナレーション形式
- **AI字幕・ナレーション自動生成をPF内に実装**
- **「15秒でPJを伝える」フォーマットの確立**

## 3. 継続投稿の障壁

### 3-1. 更新実態

- 活動報告投稿が多いほど目標到達率向上の傾向
- 経過報告が支援者同士の連帯感と達成意欲を強化
- しかし多くのPJで更新が途絶える

> 信頼度: B

### 3-2. バーンアウトの実態

- **プロクリエイターの73%**がバーンアウト症状を経験
- SNSの「常時接続」とアルゴリズムのプレッシャーが主因

**更新が途絶える主な理由:**
1. 本業との両立困難
2. 支援が集まらない時の心理的挫折
3. ネタ切れ
4. 動画制作の時間コスト
5. ネガティブコメントへの心理的負荷

> 信頼度: C（日本のクラファン起案者に特化したデータは不足）

### 3-3. 継続投稿を促す仕組み（成功事例）

| PF | 仕組み |
|-----|--------|
| Kickstarter | ダッシュボードで1時間ごとの支援額、リファラー分析等を可視化 |
| CAMPFIRE | 購入型CF + コミュニティ（月額課金型）の併用推奨 |
| 00:00 Studio（けんすう） | 創作プロセスのライブ配信PF。「見てもらうだけ」でハードル低下 |

### 3-4. PFへの示唆

- **「制作過程の共有」をデフォルトの投稿形式に**
- **投稿リマインダー + テンプレート提案**（3日間投稿なし→プッシュ通知）
- **マイルストーン自動通知**（「支援額10万円突破！」等を双方に配信）
- **「プロセスエコノミーモード」**: スマホでさっと撮った「今日の作業」を簡単投稿できるストーリーズ的軽量機能

## 4. プラットフォーム乗り換え動機

### 4-1. 移行理由

| 移行動機 | 詳細 |
|---------|------|
| 手数料の差 | CAMPFIRE 17%→10%（2021年改定）等 |
| 独自機能 | note=SEO+有料記事、FANBOX=R-18、Patreon=グローバル |
| ユーザーベースとの相性 | Makuake=製品開発者、READYFOR=社会貢献 |
| 審査の厳格さ | Makuake厳格 vs CAMPFIRE最低限 |

### 4-2. スイッチングコスト

| コスト種別 | 内容 |
|-----------|------|
| ファン基盤 | PF上のフォロワー・支援者は移行不可。**最大の障壁** |
| コンテンツ資産 | 過去の投稿・PJ履歴の移行困難 |
| 信用スコア | 成功実績・レビューが引き継げない |
| 心理的コスト | 新UIの学習、コミュニティ再構築 |

### 4-3. PFへの示唆

- 「このPFでしかできないこと」の明確化（ショート動画→ワンタップ支援→プロセス共有の一気通貫）
- 既存PFからの移行ではなく**併用を促す**ポジショニングも検討
- **過去実績のインポート機能**でスイッチングコストを下げる

## 5. 必要な支援機能

### 5-1. クリエイターが求める機能（推定ニーズ）

| カテゴリ | 機能 | 優先度 |
|---------|------|--------|
| 分析 | 動画視聴→支援転換率、リファラー分析、支援者デモグラフィック | 高 |
| リターン管理 | 支援者別進捗、発送ステータス、住所情報 | 高 |
| コミュニケーション | 一斉メッセージ、限定動画配信、コメント管理 | 高 |
| 動画制作 | テンプレート、AI字幕、BGMライブラリ、サムネイル自動生成 | 中 |
| 収益管理 | 入金スケジュール、手数料計算、確定申告用データ | 中 |
| CRM | リピート率、過去支援履歴、セグメント配信 | 低（初期） |

### 5-2. PFへの示唆

- MVP段階では**「分析ダッシュボード」と「動画テンプレート」に集中**
- リターン管理は既存ツール連携でカバー、自前構築は後回し
- 「支援速度の可視化」は動画SNSと相性良い

## 7. 初期クリエイター獲得戦略への提言

### 第1フェーズ: 西野サロン活用（ローンチ前~β期）
1. 西野本人をアドバイザー/初期起案者として確保
2. サロン内クラファン経験者を「ベータクリエイター」として招待（推定100~300名）
3. 動画制作ワークショップをサロン内で実施

### 第2フェーズ: ジャンル特化の拡大（β期~正式ローンチ）
4. CAMPFIREの手数料構造を下回る手数料で差別化
5. 「プロセス共有」特化機能で既存クラファン起案者にアプローチ
6. 特定ジャンルのコミュニティリーダーを直接獲得

### 第3フェーズ: PF成長（正式ローンチ以降）
7. 成功事例の動画コンテンツ化
8. マイクロクリエイター向け段階的ステップアップ（投げ銭→小規模CF→本格PJ）

### リスク

| リスク | 対策 |
|-------|------|
| 西野の影響力低下 | 西野以外の複数チャネルを並行構築 |
| 動画制作負荷で離脱 | テンプレート+AI支援で制作時間最小化 |
| 支援者不足で反応がない | 需要側獲得を同時進行、初期はインセンティブで確保 |
| クリエイター品質ばらつき | 最低限の審査+成功事例のフィーチャー |
