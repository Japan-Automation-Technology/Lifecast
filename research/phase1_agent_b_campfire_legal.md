# CAMPFIRE 法的構造調査レポート

**調査日**: 2026-02-14
**目的**: 「ショート動画 x クラウドファンディング」プラットフォーム設計の参考資料

---

## 1. CAMPFIREの法的スキーム

### 1.1 資金移動業登録の有無

CAMPFIREは**資金移動業者として登録していない**。[信頼度: B]

購入型・寄付型クラウドファンディングにおいて、支援者の資金をプロジェクトオーナーに移動させる行為は、本来「為替取引」として資金決済法上の資金移動業登録が必要になりうる。しかし、CAMPFIREは**収納代行スキーム（代理受領スキーム）**を採用することで、この登録義務を回避している。

### 1.2 採用スキーム: 収納代行（代理受領）

CAMPFIREの利用規約（第19条）に基づく法的スキームは以下の通り。[信頼度: A（利用規約に明記）]

1. **プロジェクトオーナーがCAMPFIREに代理受領権限を付与**
   - 利用規約第19条1項: プロジェクトオーナーは、支援金をCAMPFIREが「代わって受領するための代理受領権限」を付与する

2. **GMOペイメントゲートウェイの収納代行サービスを利用**
   - 利用規約第19条4項: プロジェクトオーナーは「GMOペイメントゲートウェイ株式会社の収納代行サービスを利用することに同意する」
   - GMO-PGにも支援金受領の権限が付与される

3. **支援者の支払い時点で債務消滅**
   - 支援者がCAMPFIRE（を通じてGMO-PG）に支払った時点で、支援者の支払義務は完了
   - その後のリターン履行責任はプロジェクトオーナーに帰属

### 1.3 法的根拠: 令和2年（2020年）資金決済法改正

2020年の資金決済法改正（法2条の2）により、収納代行スキームが為替取引規制の適用外となる条件が明確化された。[信頼度: A（法令）]

**為替取引に該当しないための要件:**

- **要件1**: 資金の受入れ時点（あるいはそれよりも前の時点）で支払者の債務が消滅する仕組みであること
- **要件2（選択肢β）**: プラットフォーム事業者が契約成立に「不可欠な関与」をしており、受取人の同意の下で資金を移動させること

CAMPFIREのスキームは以下の理由でこれらの要件を満たす:
- 支援者の支払い時点で債務が消滅する設計（要件1充足）
- CAMPFIREがプロジェクト掲載・審査・マッチングにおいて不可欠な関与をしている（要件2充足）
- プロジェクトオーナーの同意の下で資金を移動している（同意要件充足）

### 1.4 必要なライセンス・届出

| 項目 | 状態 | 信頼度 |
|------|------|--------|
| 資金移動業登録 | 不要（収納代行スキームにより回避） | B |
| 前払式支払手段発行者登録 | 該当なし | B |
| 貸金業登録 | 該当なし（融資型ではないため） | A |
| 第二種金融商品取引業 | 該当なし（投資型ではないため） | A |
| 特定商取引法の表示義務 | プロジェクトオーナーが負う | A |

## 2. 決済代行の仕組み

### 2.1 決済プロバイダー

**GMOペイメントゲートウェイ（GMO-PG）** が主要な決済インフラパートナー。[信頼度: A]

- 2013年からの長期パートナーシップ
- GMO-PGのマルチペイメントサービスを通じて12種類の決済手段を提供
- 対応決済: クレジットカード、コンビニ払い、銀行振込、PayPay、楽天ペイ、au PAY、ソフトバンクまとめて支払い等

### 2.2 資金フロー図

```
支援者 ─[支払い]──> GMO-PG (収納代行) ──[一時保管]──> CAMPFIRE ──[手数料控除後]──> プロジェクトオーナー
   |                    |                       |                          |
   |                    |                       |                          |
   ├─ クレジットカード   ├─ 収納代行として       ├─ 代理受領権限に        ├─ 支援金 - 手数料
   ├─ コンビニ払い       │  支援金を受領          │  基づき資金を保管       │  = 入金額
   ├─ 銀行振込          │                       ├─ プラットフォーム       │
   ├─ PayPay            │                       │  利用料を控除           ├─ プロジェクト成立後
   ├─ キャリア決済       │                       ├─ 決済手数料を控除       │  に入金
   └─ その他            │                       └─ 振込手数料は           └─ 送金はCAMPFIRE
                        │                          CAMPFIRE負担               指定方法で実行
                        │
                        └─ 支援者の支払い時点で
                           支援者の債務消滅
```

### 2.3 法的ポイント

- **支援者→GMO-PG**: 支援者が支払った時点で支援者の義務は消滅
- **GMO-PG→CAMPFIRE**: GMO-PGは収納代行として受領した資金をCAMPFIREに移転
- **CAMPFIRE→プロジェクトオーナー**: プロジェクト成立後、手数料を差し引いて入金
- CAMPFIREは「資金の中継」ではなく「代理受領＋収納代行」の構造で為替取引に非該当

## 3. Makuake / READYFOR との法的構造比較

### 3.1 比較表

| 項目 | CAMPFIRE | Makuake | READYFOR |

| **運営会社** | 株式会社CAMPFIRE | 株式会社マクアケ | READYFOR株式会社 |
| **上場状況** | 未上場 | 東証グロース上場（2019年） | 未上場 |
| **法的スキーム** | 収納代行（代理受領） | 収納代行（決済代行） | 収納代行（代理受領） |

| **支払い時の債務消滅** | 決済代行への支払い時 | 決済代行の承認確認時 | 決済代行への支払い時 |

[信頼度: B（各社利用規約・公開情報からの整理）]

### 3.2 各社スキームの特徴

**CAMPFIRE**
- 収納代行スキーム（GMO-PGの収納代行サービス + 代理受領権限）を利用規約で明示
- 利用規約第12条3項で「資金決済法に定める前払式支払手段」や「暗号資産に該当する」プロジェクトを禁止し、金融規制への抵触を明示的に回避
- 支援者にもシステム利用料を課す二重課金構造

**Makuake**
- CyberAgent子会社（2013年設立、2019年上場）
- 「応援購入」という用語を使用し、法的には売買契約の予約に近い構造
- 支援者資金を会社資金と別口座で管理
- 3段階の事前審査で品質管理を強化

**READYFOR**
- 日本初のクラウドファンディングサイト（2011年）
- 寄付型に強み。社会貢献プロジェクトに特化
- READYFORの法務担当者が令和2年資金決済法改正に関する詳細な法的分析を公表（noteにて）
- フルサポートプラン（17%+税）とシンプルプラン（12%+税）の2プラン体制
- キュレーターによる個別サポート体制で約75%の成功率を実現

### 3.3 共通する法的構造

3社とも以下の共通構造を採用している:

1. **収納代行スキーム**: 資金移動業登録を回避するため、決済代行会社（主にGMO-PG）の収納代行サービスを利用
2. **代理受領権限**: プロジェクトオーナーからプラットフォームに代理受領権限を付与
3. **債務消滅タイミング**: 支援者の支払い時点で債務消滅とする構造
4. **特商法の義務**: プロジェクトオーナー（販売者）が特定商取引法上の表示義務を負う
5. **免責構造**: プラットフォームはプロジェクトの成否やリターンの履行について直接責任を負わない

## 4. 過去の法的トラブル事例

### 4.1 国内の主要事例

#### 事例1: 犬の治療費募集詐欺
- **プラットフォーム**: 国内クラファンサイト（特定サイト名非公表）
- **概要**: フラットコーテッドレトリバーの心疾患治療費を募集。実際には支援金募集の2ヶ月前に犬は既に死亡
- **被害額**: 約185万円
- **結末**: 2022年1月に奈良県警が起案者を逮捕（詐欺罪）
- [信頼度: A]

#### 事例2: アイスシルクマスク詐欺
- **プラットフォーム**: 国内クラファンサイト
- **概要**: 「絹製」冷感マスクの緊急生産を謳い資金を集めたが、届いたのは化学繊維の輸入品。事後に紹介文から「絹」の文字が削除された
- **被害額**: 5,000万円以上
- **結末**: 返金要求多数も対応不透明
- [信頼度: B]

#### 事例3: みんなのクレジット（ソーシャルレンディング）
- **プラットフォーム**: みんなのクレジット
- **概要**: 融資型ファンドで40億円以上を集資。集めた資金をグループ会社への出資に流用、自転車操業状態
- **被害額**: 40億円以上（投資家への返金率約3%）
- **結末**: 2017年に金融庁・東京都から行政処分
- [信頼度: A]

#### 事例4: 和歌山市動物愛護ファンディング
- **プラットフォーム**: ガバメントクラウドファンディング（ふるさと納税型）
- **概要**: 殺処分ゼロ達成のための不妊去勢手術費用を募集したが、約1,370万円が印刷費・保険料等の目的外に使用
- **被害額**: 約2,790万円中の目的外使用分
- **結末**: 市議会厚生委員会から指摘
- [信頼度: A]

### 4.2 CAMPFIREにおけるトラブル傾向

- **リターン未送**: 「支援したのにリターンが届かない」「実行者と連絡が取れなくなった」等の報告が存在
- **高額ガジェット詐欺**: 支援総額400万円以上のプロジェクトで実行者が失踪した事例
- **CAMPFIREの姿勢**: 利用規約上、トラブルはプロジェクトオーナーと支援者間で解決すべきとし、CAMPFIRE自体は責任を負わないと明記
- **統計**: プロジェクトの9割以上は正常にリターンを完了（CAMPFIRE公称）

[信頼度: B]

## 7. 自社プラットフォーム設計への示唆

### 7.1 法的スキーム設計のポイント

1. **収納代行スキームの採用が業界標準**: CAMPFIRE・Makuake・READYFOR全社が同様のスキームを採用。新規参入でも同じ構造を取るのが安全
2. **GMO-PGとの契約が事実上のデファクト**: 3社ともGMO-PGを利用しており、同社との決済代行契約が参入障壁として低い
3. **利用規約での代理受領権限の明記が必須**: 令和2年改正法の要件を満たすため、利用規約で明確に代理受領権限の付与を規定する必要がある
4. **特商法対応はクリエイター（プロジェクトオーナー）責任**: ただし住所・電話番号の非公開設定等、プライバシー配慮も必要（CAMPFIRE 2022年対応済み）

### 7.2 リスク管理

1. **プロジェクト審査の強化**: 詐欺・不正プロジェクトの排除がブランド価値に直結
2. **免責条項の適切な設計**: ただし過度な免責は消費者保護の観点から問題になりうる
3. **消費者庁の動向監視**: 購入型クラファンへの規制強化の可能性あり
4. **2020年改正法の継続的な確認**: 収納代行スキームの適法性は解釈が変わりうる

### 7.3 手数料設計の参考

| Makuake | 20%+税 |
| READYFOR（シンプル） | 12%+税 |
