# Core Flows

## Viewer to supporter conversion flow
1. User watches video in feed.
2. User taps support.
3. App shows concise confirmation card:
   - Goal amount
   - Estimated delivery
   - Prototype status
4. User selects plan.
5. User reaches checkout page.
6. User completes payment.
7. System marks support success and updates supporter-visible state.

## Creator publishing flow
1. Complete minimum verification requirements.
2. Create project with required fields.
3. Define up to 3 plans.
4. Publish videos linked to project.
5. Monitor progress and post updates.

## Stop and appeal flow
1. Trigger stop condition.
2. Freeze/stop project and issue notice.
3. Allow explanation submission within 72 hours.
4. Resolve decision and log evidence.
